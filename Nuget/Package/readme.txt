Please note: This package is built against v3.0 of EPiServer.Personalization.Commerce. However the package has 
a number of updates since v3.0 but there is no assembly redirect as for most Episerver assemblies.

So I recommend adding an assembly redirect into web.config as follows:

<configuration>
  <runtime>
    <assemblyBinding xmlns="urn:schemas-microsoft-com:asm.v1">
      <dependentAssembly>
        <assemblyIdentity name="EPiServer.Personalization.Commerce" publicKeyToken="8fe83dea738b45b7" culture="neutral"/>
        <bindingRedirect oldVersion="0.0.0.0-3.0.7.0" newVersion="3.0.7.0"/>
      </dependentAssembly>
    </assemblyBinding>
  </runtime>
</configuration>

The example redirects to v3.0.7 which is the current released version of EPiServer.Personalization.Commerce at the time of publishing
